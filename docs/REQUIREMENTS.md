# 原始题目要求（48h全栈挑战）

> 本文件是题目原文的忠实提取，用于开发过程中对照验证。设计方案见 PRD_BiliDigest.md。

---

## 用户痛点

很多B站长视频没时间看，但用户需要：
- 快速掌握视频的关键事实、论点、逻辑链条、结论与可行动建议
- 以后可以用不同用途/风格再次生成内容（而不是一次性总结完就丢）

## 任务目标

做一个系统（形态不限：webapp/APP/插件/桌面均可），输入一个无付费墙的B站视频URL，产出并沉淀"可复用的知识资产"，并支持基于该资产进行多种输出生成与再利用。

---

## 必须包含的四大能力模块

### 模块1：输入与资产化

- 输入：任意公开可访问的B站视频URL
- 可用API：
  - `https://api.bilibili.com/x/web-interface/view` 获取视频信息
  - `https://api.bilibili.com/x/player/playurl` 获取视频流地址
- 系统需要把该视频变成一个"资产"，后续可以再次被访问/复用/再生成
- **留白：** 具体怎么组织、如何判断"复用"、如何更新，全部由开发者设计

### 模块2：内容抽取

- 从视频中抽取"足以支撑后续生成任务"的信息
- **硬性要求：必须以某种形式包含视觉内容，不能纯音频和转写/字幕**
- 可抽取的内容示例（但不限于）：
  - metadata（标题/简介/UP/标签等）
  - 音频/字幕/转写文本
  - 关键帧/截图/视频片段
  - 任何其他必要信息（例如：论点列表、时间线、概念表、因果链等）
- **留白：** 抽什么、抽多少、按什么策略抽，各类信息先抽再存还是先存再抽，怎么存，都是设计的一部分

### 模块3：记忆化存储（Memory/RAG/DB任意）

- 把"抽取结果 + 结构化/向量化 + 引用/溯源能力"写入一个存储层
- 必须支持：
  - 回看某个视频的资产
  - 基于一个或多个视频进行特定信息查询/生成
- **留白：** 可以选mirix.io/mem0之类的开源memory框架，也可以用md/json/SQL/RAG手搓；选型与数据结构是考察点

### 模块4：输出层（未来可扩展）

- a) 用户可以通过选项指定输出模态
- b) 用户还可以通过自然语言prompt指定生成内容的侧重点（偏事实、偏观点、偏步骤、偏争议、偏复盘）
- c) 用户可以指定一个或多个视频资产作为内容来源
- d) **至少实现2种输出模态**，可从以下选择（也可自创）：
  - 图文并茂总结（截图/关键帧做配图）
  - 小红书图文（标题、分段、tag、风格化表达）
  - 理解测验题集：输出一套题来检验观众是否真正理解视频
  - 脑图/结构图：把视频知识结构化成可视化

## 交付要求

- 一个GitHub repo + README（能让评审在本地跑起来/看到效果）
- （可选）Demo录屏或可访问链接
- 一页简短说明（可写在README里）：
  - 你的核心取舍：抽取策略、memory结构、输出抽象、你砍掉了什么
  - 失败/不确定性处理：缺字幕、下载失败、模型不稳定等怎么降级
  - 如果再给你1周，你会怎么迭代

---

## 评审考察维度

1. **产品感：** 是否有"用户中心"思维、输出是否"真的有用"、结构是否抓住关键点
2. **现实主义的美感：** 在工程可实现的前提下，用最简化的设计达到高级感与可用性
3. **工程能力：** 交付是否可运行、结构是否清晰、是否可扩展
4. **信息检索和快速学习：** 可参考项目、可用工具、API、debug等
5. **节奏感：** 开发规划、迭代次数、每次迭代的"反馈→改动→效果"链条
6. **新工具灵敏度：** 选型与上手是否高效、如何取舍
7. **使用AI开发能力：** 能否充分借助AI在48h内完成远超手搓规模的工程
8. **承压与取舍：** 是否形成闭环、是否能在复杂度里做正确删减

## 开发自检清单（对照评审标准）

在提交前，用以下问题自检：

- [ ] 输入一个B站URL，系统能自动完成全部处理吗？（工程闭环）
- [ ] 输出中是否包含视觉内容（关键帧/截图）？（硬性要求）
- [ ] 是否实现了至少2种输出模态？（硬性要求）
- [ ] 用户能否通过自然语言prompt改变输出侧重点？（模块4-b要求）
- [ ] 用户能否选择多个视频资产生成输出？（模块4-c要求）
- [ ] 存储层是否支持"回看"和"查询"？（模块3要求）
- [ ] 生成的内容能否溯源到视频的具体时间点？（引用/溯源能力）
- [ ] 评审clone后能按README跑起来吗？（交付底线）
- [ ] README是否包含取舍说明、降级策略、未来迭代？（交付要求）
- [ ] 缺字幕、下载失败、LLM出错时系统会怎样？（降级处理）
